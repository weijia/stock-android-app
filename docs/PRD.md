# 股票实时行情 Android 应用 - 产品需求文档

## 1. 需求背景

### 1.1 业务背景
用户需要一个轻量级的 Android 应用来实时监控股票价格走势。该应用需要从已有的股票价格 HTTP 服务获取数据，并以直观的图表形式展示价格曲线和成交量信息。

### 1.2 用户痛点
- 现有股票行情应用功能繁杂，加载缓慢，对老旧设备兼容性差
- 无法自定义数据源，依赖第三方平台的数据更新频率
- 缺乏简洁直观的价格走势可视化

### 1.3 解决方案价值
开发一个纯 Java 实现的轻量级 Android 应用，支持最老版本的 Android 系统（API Level 14+），每分钟自动从用户自建的股票价格服务器获取数据，并以图表形式展示价格走势和成交量。

## 2. 目标与范围

### 2.1 产品目标
| 目标类型 | 具体目标 |
|---------|---------|
| 核心目标 | 提供简洁、稳定的股票实时行情展示功能 |
| 兼容性目标 | 支持 Android 4.0 (API 14) 及以上所有版本 |
| 性能目标 | 应用启动时间 < 2秒，内存占用 < 50MB |

### 2.2 MVP 范围
- 股票代码输入与查询
- 实时价格展示（当前价、涨跌幅、成交量等）
- 价格走势折线图
- 成交量柱状图
- 每分钟自动刷新
- 服务器地址配置

### 2.3 后续迭代规划
| 阶段 | 功能 | 优先级 |
|-----|------|-------|
| V1.1 | 多股票监控列表、本地缓存 | 中 |
| V1.2 | 价格预警通知、历史数据查看 | 低 |
| V2.0 | 趋势分析展示、深色主题 | 低 |

## 3. 用户与场景

### 3.1 目标用户
| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 个人投资者 | 持有少量股票，关注实时价格变动 | 快速查看持仓股票价格走势 |
| 老旧设备用户 | 使用 Android 4.x/5.x 设备，无法安装主流应用 | 兼容性好的轻量级应用 |

### 3.2 核心使用场景
1. 开盘时段监控：用户在交易时段打开应用，实时观察股票价格波动
2. 快速查询：用户输入股票代码，快速获取当前行情
3. 走势分析：用户通过图表判断近期价格走势和成交量变化

## 4. 功能需求

### 4.1 功能概览表
| # | 模块 | 功能描述 |
|---|------|---------|
| 1 | 股票查询 | 用户输入股票代码（如 000001），应用从服务器获取实时行情数据并展示 |
| 2 | 实时行情展示 | 展示股票名称、当前价格、涨跌幅、涨跌额、成交量、成交额等核心数据 |
| 3 | 价格走势图 | 以折线图形式展示最近 N 天的收盘价走势，支持触摸查看具体数值 |
| 4 | 成交量图 | 以柱状图形式展示最近 N 天的成交量，与价格走势图联动展示 |
| 5 | 自动刷新 | 每隔 60 秒自动从服务器获取最新数据，刷新界面展示 |
| 6 | 服务器配置 | 用户可配置服务器地址（默认 localhost:8080），支持修改端口和 IP |

### 4.2 模块详情

#### 模块 1：股票查询
- 业务逻辑：用户输入 6 位股票代码 → 点击查询按钮 → 应用调用 `/api/realtime/{code}` 接口 → 解析返回数据 → 更新界面
- 交互逻辑：输入框支持数字键盘，代码格式校验（必须为 6 位数字），查询按钮点击后显示加载状态
- 规则约束：股票代码必须为 6 位数字；查询失败时显示错误提示；网络超时设置为 10 秒

#### 模块 2：实时行情展示
- 业务逻辑：解析 API 返回的 JSON 数据，提取 name、price、change_pct、volume 等字段，格式化后展示
- 展示规则：
  - 涨跌幅 > 0 时价格和涨跌幅显示绿色（中国股市惯例）
  - 涨跌幅 < 0 时显示红色
  - 成交量单位转换为"万手"，成交额转换为"万元"

#### 模块 3 & 4：价格走势图与成交量图
- 业务逻辑：调用 `/api/kline/{code}?days=30` 获取历史 K 线数据，解析 data 数组中的 date、close、volume 字段
- 图表规则：
  - 价格走势图：折线图，X 轴为日期，Y 轴为收盘价
  - 成交量图：柱状图，X 轴与价格图对齐，Y 轴为成交量
  - 支持触摸/点击查看具体日期的数据值
  - 图表使用 Canvas 绘制，不依赖第三方图表库

#### 模块 5：自动刷新
- 业务逻辑：应用启动后台定时任务，每 60 秒调用一次 `/api/realtime/{code}` 接口获取最新数据
- 交互逻辑：刷新时界面顶部显示"正在更新..."提示；刷新完成后提示消失；刷新失败显示错误信息
- 规则约束：
  - 刷新间隔固定为 60 秒，不可配置
  - 应用退出时停止刷新任务
  - 网络不可用时跳过本次刷新，不中断定时器

#### 模块 6：服务器配置
- 业务逻辑：用户可修改服务器 IP 和端口，配置保存到本地 SharedPreferences，下次启动自动读取
- 默认值：服务器地址默认为 localhost，端口默认为 8080

## 5. 数据与接口

### 5.1 API 接口依赖
| 接口 | 用途 | 调用时机 |
|-----|------|---------|
| `/api/realtime/{code}` | 获取单只股票实时行情 | 用户查询、自动刷新 |
| `/api/kline/{code}?days=30` | 获取历史 K 线数据 | 用户查询时获取图表数据 |
| `/api/health` | 检查服务器状态 | 应用启动时检测连接 |

### 5.2 数据字段映射
| API 字段 | 应用展示 | 格式化规则 |
|---------|---------|---------|
| name | 股票名称 | 直接展示 |
| price | 当前价格 | 保留 2 位小数 |
| change_pct | 涨跌幅 | 保留 2 位小数，添加 % 符号 |
| change_amt | 涨跌额 | 保留 2 位小数 |
| volume | 成交量 | 转换为万手单位 |
| amount | 成交额 | 转换为万元单位 |

## 6. 技术约束

关键技术约束：
- 开发语言：纯 Java，不使用 Kotlin
- 最低兼容版本：Android 4.0 (API Level 14)
- 图表实现：使用 Android Canvas 自绘，不依赖第三方图表库
- 网络请求：使用 HttpURLConnection（原生 API，无需第三方库）
- JSON 解析：可使用轻量级库或手动解析
- 定时任务：使用 Handler + Runnable 或 Timer
